Growth

Reflection:

Before starting the HTML/CSS project, our last CS project before CAP, I had already been used to coding on code.org's app lab and game lab, so transitioning to this sort of coding was deemed new and quite difficult for me. I wasn't at all familiar with HTML/CSS, so learning to incorporate all of its elements into code, with all its ups and downs, represents an area of growth for me. I was most unfamiliar with using text to code, instead of simply dragging blocks. And once I understood these fundamentals, more difficult HTML components kept piling up.

The hardest areas for me to master was having to control the images' locations and style CSS alterations. Since I was used to using an X/Y grid box for navigating an image's location or dragging the images to the location I wanted it to go to in App Lab, the transition to utilizing and mastering the style CSS way for moving images was difficult, like the use of text-wrap, text-align, order, and etc. Image alterations in the CSS style sheet were difficult for me, and I was able to get past these obstacles through referencing back to my past HTML assignments and constatnly doing trial-and-error.

I was frequently aiming to make my website as appealing to the audience and aesthetic as possible, so in class, I devoted a lot of my time to images that would match the theme and so forth, along with balancing the actual coding component of the website. And in devoting so much time to making my website as fucntional and aesthetic as possible, my website ended up looking satisfactory and functioned successfully.

Knowing what I know now, I would've probably added more HTML pages and a navigation tab on the top of the website for easier navigation and access for the users.
